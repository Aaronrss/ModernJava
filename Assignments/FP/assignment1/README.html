Java Object-Oriented and Functional Array (Assignment 1)

The objective of this assignment is to give you experience using Android Studio to develop and test an implementation of an Array interface written using Java's object-oriented and functional programming features. It's important that your Array implementation work properly since it is required by all subsequent assignments in this course.

The following resources may be helpful in completing this assignment:

  • java.utils.Arrays documentation, which provides certain useful methods for your Array implementation.
  • The Oracle Java Tutorial, which provides a good overview of Java's object-oriented features.

If your Java knowledge is rusty you might consider watching the videos in our Coursera Java for Android MOOC.

The assignment is packaged as a project using IntelliJ and is written using modern Java. For the purposes of Assignment 1 you need to be aware of the following directories:

Program Description and "TODO" Tasks

An Array is an abstract data type (ADT) implemented via a single contiguous buffer that provides operations like get(), set(), add(), remove(), size(), replaceAll(), and forEach(). Your task is to implement a generic Array class in Java. This class will be different from Java built-in arrays in various ways, e.g., it can be resized and used in conjunction with other Java Collection Framework classes.

You'll need to modify several files containing the skeleton Java code by completing the "TODO - you fill in here" tasks to provide a working solution. Do not change the overall structure of the skeleton - just fill in the "TODO" tasks. In particular, you'll need to finish implementing the following "TODO" tasks for this assignment in the ... folder:

  • Array.java. You'll need to complete the "TODO" tasks in various methods to implement the Array class.
    • Implement the requirements of the Iterator interface.

    You should avoid using Java loops in this assignment as much as possible and instead use Arrays.copyOf() and/or System.arraycopy().

    Unit Tests

    Unit tests in the ... folder are provided to check that your implementation is working as expected. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the unit tests to detect problems in your code. However, make sure to run these tests prior to submitting your solution and seek help on the discussion forum if you have questions about why any of the tests fail.
